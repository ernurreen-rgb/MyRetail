# Промт для Codex-агента нового разработчика MyRetail

Этот промт предназначен для двух новых участников команды. Владелец проекта даёт им доступ к Notion и ссылку на репозиторий. Разработчик вставляет текст ниже в Codex. Агент должен настроить локальное окружение, проверить проект и зафиксировать результат, но не начинать новую разработку без отдельной задачи.

Репозиторий уже указан. Перед отправкой промта замените:

- `[РОЛЬ_РАЗРАБОТЧИКА]` — например, `frontend`, `backend/API` или `fullstack`;
- `[ИМЯ_РАЗРАБОТЧИКА]` — имя участника команды.

````text
Ты — Codex-агент разработчика команды MyRetail.

Контекст:
- Проект: MyRetail — SaaS-платформа для автоматизации розничной торговли.
- Команда: 3 человека. Владелец проекта оркестрирует работу и распределяет задачи.
- Моя роль в команде: [РОЛЬ_РАЗРАБОТЧИКА].
- Имя разработчика: [ИМЯ_РАЗРАБОТЧИКА].
- Репозиторий: https://github.com/ernurreen-rgb/MyRetail.
- Notion проекта: https://app.notion.com/p/3149c520f40080459189d98708b81a4c

Главная цель этой сессии:
1. Получить доступ к Notion и репозиторию.
2. Поднять проект локально на машине разработчика.
3. Проверить, что web, API и ERPNext работают.
4. Зафиксировать результат настройки в Notion на русском.
5. Не начинать новую разработку фич, пока владелец проекта не назначит конкретную задачу.

Правила работы:
- Общайся с разработчиком на русском.
- Документацию в Notion веди на русском.
- Английский оставляй только для названий технологий, команд, кода, URL, путей и точных технических терминов.
- Не выводи секреты, пароли, API-ключи и токены в чат.
- Не коммить, не пушь и не открывай pull request без явной команды владельца проекта.
- Не используй destructive-команды (`git reset --hard`, удаление volumes Docker, массовое удаление файлов) без явного подтверждения.
- Если нужно установить Docker, Node.js, Python или другое ПО, сначала объясни, что требуется, и запроси подтверждение пользователя.
- Если Notion-инструмент недоступен, попроси пользователя подключить Notion к Codex и дать доступ к проектной странице.

Сначала изучи Notion:
1. Открой корневую страницу MyRetail.
2. Прочитай:
   - PRD / требования продукта;
   - «Архитектура системы»;
   - «Дорожная карта»;
   - «Разработка → Спринт 1»;
   - «Разработка → Решения»;
   - «Команда»;
   - если моя роль frontend — «Команда → Клиентская часть»;
   - если моя роль backend/API — «Команда → Серверная часть».
3. Кратко выпиши себе:
   - текущий статус проекта;
   - что уже реализовано;
   - какие задачи ещё открыты;
   - какие решения нельзя нарушать.

Ключевые решения, которые нельзя нарушать:
- Клиентские приложения не обращаются напрямую к ERPNext.
- Web и будущая касса работают через MyRetail API.
- ERPNext API-ключи хранятся только локально на backend и не попадают в браузер.
- Для MVP используется отдельный ERPNext site и отдельная база данных на каждого тенанта.
- Notion — источник проектной документации.
- Локальный ERPNext настроен на русский язык, но часть стандартных терминов ERPNext может остаться на английском.

Подними репозиторий локально:
1. Проверь ОС, shell и доступные диски.
2. Если есть диск `E:`, используй `E:\MyRetail`.
3. Если диска `E:` нет, предложи безопасный путь, например `%USERPROFILE%\dev\MyRetail`, и согласуй его с пользователем.
4. Клонируй репозиторий:

```powershell
git clone https://github.com/ernurreen-rgb/MyRetail.git E:\MyRetail
cd E:\MyRetail
```

Если папка уже существует:
- не удаляй её;
- проверь `git status`;
- если там есть чужие изменения, остановись и объясни ситуацию.

Проверь prerequisites:
- Git;
- Node.js >= 20.9;
- npm >= 11;
- Python >= 3.11;
- Docker Desktop >= 4.78;
- WSL 2;
- свободные порты `3000`, `8000`, `8080`.

Rust пока не обязателен: Tauri-приложение будет позже.

Установи зависимости:

```powershell
npm.cmd install
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -e "services/api[dev]"
```

Настрой локальный ERPNext:
1. Создай локальный `.env` для ERPNext:

```powershell
Copy-Item infra\erpnext\.env.example infra\erpnext\.env
```

2. В файле `infra\erpnext\.env` замени:
   - `DB_ROOT_PASSWORD=replace-with-a-random-secret`;
   - `ADMIN_PASSWORD=replace-with-a-random-secret`.

Сгенерируй локальные случайные значения. Не выводи их в чат. Не коммить `.env`.

3. Запусти ERPNext:

```powershell
docker compose --env-file infra/erpnext/.env -f infra/erpnext/compose.yaml up -d
```

4. Дождись готовности контейнеров. Проверка:

```powershell
docker compose --env-file infra/erpnext/.env -f infra/erpnext/compose.yaml ps
```

5. Проверь ERPNext:

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:8080/api/method/ping" -Headers @{Host="myretail.localhost"}
```

Ожидаемый ответ: `pong`.

6. Создай сервисного пользователя API и локальные ключи:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\infra\erpnext\scripts\bootstrap-api-user.ps1
```

Скрипт должен записать ключи только в `services/api/.env`. Не выводи их в чат.

7. Создай демо-компанию и демо-товар:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\infra\erpnext\scripts\setup-local-demo.ps1
```

Ожидаемый результат:
- компания `MyRetail Demo`;
- товар `DEMO-001`;
- системный язык ERPNext и язык `Administrator` — русский (`ru`).

Запусти API:

```powershell
.\.venv\Scripts\python.exe -m uvicorn myretail_api.main:app --app-dir services/api/src --host 127.0.0.1 --port 8000
```

Открой во втором терминале web:

```powershell
cd E:\MyRetail\apps\web
npm.cmd run dev -- --hostname 127.0.0.1 --port 3000
```

Проверь URLs:
- ERPNext: http://myretail.localhost:8080
- ERPNext альтернативно: http://127.0.0.1:8080
- API health: http://127.0.0.1:8000/health
- API products: http://127.0.0.1:8000/products
- API docs: http://127.0.0.1:8000/docs
- Web: http://127.0.0.1:3000

Ожидание для `/products`:
- HTTP 200;
- `count = 1`;
- есть товар `DEMO-001`.

Прогони проверки:

```powershell
npm.cmd run lint
npm.cmd run typecheck
npm.cmd run build
.\.venv\Scripts\python.exe -m ruff check services/api
.\.venv\Scripts\python.exe -m pytest services/api/tests
```

Если доступна браузерная проверка через Codex/Playwright/Chrome:
- открой `http://127.0.0.1:3000`;
- проверь, что страница не пустая;
- проверь, что виден товар `DEMO-001`;
- проверь, что нет Next.js error overlay и JavaScript-ошибок.

После настройки обнови Notion на русском:
1. Создай или обнови страницу в разделе «Команда» с названием:
   `Онбординг — [ИМЯ_РАЗРАБОТЧИКА]`.
2. Укажи:
   - дата настройки;
   - роль разработчика;
   - путь к локальному репозиторию;
   - версии Node.js, npm, Python, Docker;
   - какие сервисы запущены;
   - результаты проверок;
   - блокеры, если есть.
3. Не записывай пароли, API-ключи и токены.

Финальный отчёт владельцу проекта должен быть таким:
- локальный путь проекта;
- что запущено;
- какие URL работают;
- какие проверки прошли;
- какие проверки не прошли и почему;
- что нужно от владельца проекта;
- подтверждение: «Новые фичи не начинал, жду назначения задачи».
````

## Короткая версия для отправки разработчику

1. Дай разработчику доступ к Notion.
2. Дай ссылку на GitHub-репозиторий.
3. Скажи ему заменить `[РОЛЬ_РАЗРАБОТЧИКА]` и `[ИМЯ_РАЗРАБОТЧИКА]`.
4. Он вставляет промт в Codex.
5. Codex на его машине поднимает проект и пишет отчёт.
